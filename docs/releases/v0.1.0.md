# Release Notes — v0.1.0 (Local Release Candidate)

## Project

PSR-SRS MVP — E-commerce Personalized Search Ranking & Semantic Retrieval

## Overview

This is the **initial local Release Candidate** of the MVP. It demonstrates
a complete pipeline from synthetic data generation through multi-channel
retrieval, user profiling, personalized re-ranking, and offline evaluation —
all running locally with no external services.

## What's Included

### Data Pipeline

- Configurable synthetic data generator (500 items, 100 users, 200 queries)
- Cascade browsing behavior model with realistic e-commerce funnel
- 66 automated data quality checks
- Dual-run reproducibility verification (5/5 SHA-256 match)

### Retrieval

- BM25 keyword retrieval (Okapi, k1=1.5, b=0.75)
- LSA semantic retrieval (TF-IDF + TruncatedSVD, 64-d, 93.3% explained variance)
- Linear Hybrid fusion (0.5/0.5, min-max normalization)

### Personalization

- User profiles from weighted behavior + time decay
- Category/brand/price affinity scoring
- Cold-start fallback (100% exact match)

### Evaluation

- Precision@K, Recall@K, MRR@K, NDCG@K (qrels-based)
- HitRate@K, Behavioral MRR/NDCG, Positive Recall
- Candidate positive coverage diagnostics

### Notebook

- 44 cells, 18 chapters, end-to-end pipeline demonstration
- Cache mode (pre-computed outputs) and recompute mode (full rebuild)
- Both modes verified: 0 errors, all cells executed

## Known Limitations

- **Synthetic data only** — not representative of real user behavior
- Small scale (500 items, 100 users)
- BM25 vocabulary limited to 76 terms
- Candidate positive coverage only 13.85%
- Fixed weights (no Learning to Rank)
- No online A/B test capability
- No production deployment

## What This Release Is Not

- Not a production-ready search system
- Not an online service or API
- Not a replacement for Enterprise-level infrastructure
- Not a business performance claim
